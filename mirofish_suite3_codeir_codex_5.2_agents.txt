    | 
    | - `backend/app/utils/llm_client.py:66` used to do `re.sub(r'<think>...'</think>', '', content).strip()`.     | - That empty content then flows into section saving: `backend/app/services/report_agent.py:2094` writes `## {section.title}` and only appends body if `cleaned_content` is non-empty, so you end up with a super thin section.
    | - `_clean_section_content` can also make the “thin” outcome worse by removing a duplicated title line at the top of the model output, potentially turning “title-only” outputs into fully empty bodies: `backend/app/services/report_agent.py:2131`.
    | 
    | I patched `backend/app/utils/llm_client.py:17` so `<think>` stripping is “lossless”: it removes think blocks when there’s real content outside them, but if removing them would make the output empty, it keeps the inner text (removes only the tags).     | That “last session is missing / partly missing” is coming from how we write activities into the Zep graph.
    | 
    | `ZepGraphMemoryUpdater` buffers activities per platform and only sends them when the buffer hits `BATCH_SIZE` (5). If the simulation goes idle/“finished” with fewer than 5 activities left in the buffer, that tail batch doesn’t get sent until the updater is stopped (e.g., when the sim process fully exits), so searches immediately after “finish” can’t see the last chunk yet. See `backend/app/services/zep_graph_memory_updater.py:365`.  
    | 
    | I fixed this by adding an idle-time auto-flush: when the queue is quiet for `FLUSH_INTERVAL` (2s), we now flush any remaining buffered activities so the last session shows up quickly in search.
